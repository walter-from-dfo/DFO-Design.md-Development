01
Why this exists

Lock the source, free the output

Without structure, every carousel image is bespoke — quality drifts, decisions get re-litigated, and we can't audit why an output said what it said. The Primitives Framework is the fix.

"

The core principle. The quality of any AI-generated asset is bounded by the quality of its source material. Lock the source material, and generation becomes the cheap step. Change a primitive, and every downstream asset regenerates correctly — automatically.

Reproducibility

Same bundle + same skill = same shape of output every time. No more re-litigating decisions.

Auditability

Every L4 asset traces back to its primitive sources. Wrong claim? Trace it to L1.

Speed

Locked primitives turn generation into the cheap, fast step in the workflow.

Scale

A new vertical is a new named positioning — not a new everything.

03
Layer by layer

What lives where, and why

Full file lists, rules, and responsibilities for each layer — plus gap analysis for where the system needs to grow.

L0
Evidence
Raw source materials — never edited, never interpreted
L0 is the product team's working materials. Everything here is a primary source. Nothing in L0 is an interpretation — it's a fact or a file. If a hex value appears in Color_Palette.svg, that's L0. The decision to use that color as the accent for a product line is L1.
Brand L0 files
  • Logo.svg Master wordmark in all 4 colorway treatments. The gold sparkle (#f2c11e) is canonical here.
  • Color_Palette.svg Every hex value in the system. Source of truth for all accent colors.
  • NeuzeitGrotesk-ExtraCondBlack.ttf The header font. The single most recognizable typographic element.
  • Poppins-SemiBold.ttf (plus other weights) Body and UI font for all templates.
  • Shapes.svg Nine wave path variants, all in #a8cae4.
  • Sparkles_1–11.png Per-palette sparkle cluster images.
Product L0 files
  • Clean product photos — cutout or near-cutout on neutral background
  • In-use / lifestyle photos — product on body (foot, hand, etc.)
  • Before / after customer photos — honest condition shots, real skin
  • Doctor portrait — professional, lab coat, warm pose
  • Doctor's handwritten signature scan
  • Reference carousel images (01–11) — completed examples attached to every generation call
  • COAs, spec sheets, supplier declarations (if claims require verification)
!
Gap — no formal L0 inventory There is no master manifest of L0 files per SKU. If a product photo gets overwritten or a reference image goes missing, there's no audit trail. Recommendation: maintain a simple L0 manifest per product (filename, date added, source) so generation calls can always verify their inputs exist and are current.
L1
Primitives
Immutable brand truth — facts derived from L0
L1 is what you extract from L0 and lock. It does not change unless the brand itself changes. An agent loading L1 knows everything it needs to produce brand-consistent output — without ever seeing an L0 file. These three files live permanently in the Claude Project and are inherited by every conversation automatically.
global-rules.md
  • Two-font stack — Neuzeit Grotesk ExtCond Black (headers) + Poppins (everything else)
  • Two-tone header rule — mandatory at 140pt+, no exceptions
  • Sparkle anatomy — 3-sparkle cluster config, 2-color rule, placement rules
  • Canvas spec — 2000×2000px square, mobile-first at 375px
  • Seven layout principles (product always visible, one message per image, etc.)
  • Logo colorway rules — which version to use on which background
  • Wave shape placement rules
color-system.md + reusable-elements.md
  • All fixed hex values — #1d396a, #fef5e3, #a8cae4 — never change across SKUs
  • Six product-line accent palettes with primary, secondary, and support hex values
  • Dark background rule — what changes when background is navy
  • Wave shape — 9 variants, color rules, placement rules
  • Pop-out circle detail — anatomy, outline color, arrow matching rule
  • Badge construction — white icon on complementary color, 4–6 max
  • Connecting arrow color rule — must inherit exact header accent hex
  • Authority signature block — authenticity rules for doctor's signature
  • Day-count badge spec — pill shape, accent fill, specificity rule
!
Gap — pink/rose palette not fully documented The pink/rose accent palette (used for Fuzzy Heel Socks) is noted as product-specific and not extracted into color-system.md. The designer must supply the hex at brief time. This should be formalized as a named palette entry with primary, secondary, and sparkle-support hex values locked.
L2
Locked Judgment
Where valid L1 interpretations get picked once and locked
L2 is the decision layer. L1 defines the rules; L2 makes the calls. The 11 template .md files are the core of L2 — each one locks a carousel slot's layout variant choices, element requirements, and review criteria. Named positionings (audience-specific variants) live at L2. So does the competitive framing for comparison charts.
What each template .md file locks
  • Which layout variants are available and when to choose each
  • Every required design element and its placement rule
  • Typography sizes and weight roles specific to that template
  • Color guidance for that template's zones
  • Common mistakes to avoid — the negative spec
  • The complete variable list — every placeholder that must be filled
  • What "correct" looks like — the review criteria
Named positionings (template variants)
  • Template 09 — Before/After vs. Product Benefit vs. Multi-Problem Grid
  • Template 08 — Light background vs. Dark background variant
  • Template 06 — How It Works vs. How To Use vs. How To Wear
  • Template 11 — Gifting variant vs. Product Family variant
  • Template 03 — Light header block vs. Deep navy header block
  • All variants inherit the base rules from global-rules.md (L1)
!
Gap — brand L2 is implicit, not documented Brand voice, anti-positioning, and reserved language currently live in people's heads. There is no brand-positioning.md file at L2. An agent could produce technically correct output (L1 rules followed) that is brand-wrong (wrong tone, wrong claim tier). Recommendation: create a one-page brand-positioning.md that locks voice, anti-positioning, and claim hierarchy.
!
Gap — no product-line spec file per SKU family Which of the 11 slots are active for each product line, and which variants are chosen, is decided per-project rather than locked in a file. A one-page sku-family-spec.md per product line would make L3 fully deterministic.
L3
Production
A four-step sequence that turns primitives into assets
L3 never invents — it renders. Every action here is downstream of a locked primitive bundle (L1 + L2 from all three pyramids). The four roles below are not alternatives; they run in order — bundle assembled, prompt filled, image rendered, output reviewed.
  1. 1
    BundleHuman or Claude Project
    Assemble the primitive bundle

    Pull L1 + L2 from all three pyramids for the one slot being built. Attach the reference carousel image + the product photo. Never load all 11 template files at once.

    • Brand — global-rules.md · color-system.md · reusable-elements.md
    • Product Line — accent hex values · active slot list · variant rules
    • Product — product fact sheet · filled prompt template · headline copy
  2. 2
    JudgmentClaude
    Fill DFO_Prompt_Templates.md

    Claude reads the bundle, chooses the layout variant, writes the two-tone headline split, and replaces every {{PLACEHOLDER}} in the prompt for that single slot. Output is a complete, generator-ready prompt.

  3. 3
    RenderGPT Image 2 · Nano Banana
    Generate the L4 image

    The image generator receives the filled prompt + reference image + product photo and renders the carousel slot. No interpretation — what was specified is what gets drawn.

  4. 4
    ReviewHuman designer
    Run the three Judges

    Truth · Alignment · Brand. Approve → ship to L4. Fail → return to step 1 or 2 with refinement notes. Never patch the output — fix the primitive that caused the failure.

!
Gap — no automated pipeline yet L3 currently runs manually. The design-brain-action-plan.md outlines an n8n workflow (4 nodes: receive → LLM prompt → generate → save) but it hasn't been built yet. Recommendation: build only after manual runs are reliable across all 11 template types. Automating a broken process makes failures faster.
L4
Generated Assets
Finished carousel images — disposable, regenerated as primitives evolve
L4 is the only layer that is meant to change. These are the outputs, not the source of truth. When a primitive changes (new product photo, updated accent palette, revised benefit claim), L4 is regenerated — not edited in place. Editing L4 directly is always wrong; it severs the audit trail.
The 11 carousel outputs
  • 01_Feature_Dominant — lead image, product dominant
  • 02_Comparison_Chart — competitive framing
  • 03_Use_Case_A — lifestyle / context, multi-scenario
  • 04_Product_Details — what's in the box, dimensions
  • 05_Use_Case_B — single item feature callout
  • 06_How_It_Works — step-by-step application
  • 07_Sizing_Chart — measurement guide
  • 08_Doctor_Developed — physician credibility
  • 09_Problem_Solution — before/after, condition coverage
  • 10_Social_Proof — press logos, earned media
  • 11_Cross_Sell — gifting or product family
Auditability requirements
  • Each L4 file should record which primitive bundle produced it
  • Wrong claim in output → trace to L1 product fact sheet
  • Wrong layout → trace to L2 template .md file
  • Wrong color → trace to L1 color-system.md accent palette
  • Wrong variant chosen → trace to L2 SKU family spec
  • Never edit L4 directly — fix the primitive and regenerate

How the bundle composes. When an agent builds carousel slot 9 for SKU X, it pulls L1 + L2 from all three pyramids simultaneously — brand rules + product-line accent + product brief — and hands the combined bundle to L3. No pyramid is optional. Missing one means the output is either brand-wrong, palette-wrong, or product-wrong.

01

Feature Dominant

Lead image — make the product look so good the customer doesn't need to ask questions. Two zones: typography + callouts left, product right.
Typography
  • Header Neuzeit Grotesk ExtCond Black Two-tone — e.g., "BLISTER" (navy) + "BE GONE" (accent)194pt
  • Feature label Poppins SemiBold In a dark contrast block52pt
  • Descriptor Poppins Medium Lighter weight, supports the label42pt
  • Disclaimer Poppins Medium24pt
Common mistakes
  • Product too small. If you read the callout before noticing the product, scale it up.
  • More than 3 callouts. 4+ starts to feel like a spec sheet. Cut to the differentiating benefits.
  • Drop shadow missing. Any product close in tone to the background disappears without it.
  • Accent color overused. Reserve it for header emphasis, wave, and sparkles — not the callout blocks too.
02

Comparison Chart

Visual argument — brand column saturated and dominant, competitor columns receding. The conclusion is foregone before you read a row.
Typography
  • Header Neuzeit Grotesk ExtCond Black Multi-line stack, navy on cream — "DESIGN THAT'S STEPS AHEAD"194pt
  • DFO column title Poppins SemiBold Mixed-case "Dr. Frederick's" — softer, human, signals the hero column42pt
  • Competitor column titles Neuzeit Grotesk ExtCond Black ALL CAPS — "REGULAR BANDAGES", "OTHER BRANDS". Different font from DFO column on purpose.42pt
  • Row label (main) Poppins SemiBold36pt
  • Row sub (supporting line) Poppins Medium24pt
  • Disclaimer Poppins Regular — always present24pt
Common mistakes
  • Equal visual weight. The brand must win before the customer reads a row. Wider, taller, more saturated.
  • Named competitors. Never use real brand names — "Regular Bandages," "Other Brands," "Copycat" only.
  • More than 10 rows. 6–8 is the sweet spot. Anything more breaks at mobile size.
  • Typographic checkmarks. Use filled circles with white checks — not ✓ characters.
  • No disclaimer. Always include the comparison methodology fine print.
03

Use Case A — Lifestyle

Context image. Where the product fits in real life. Multi-zone body grid + lifestyle photo. The only template that permits a 3-color sparkle.
Typography
  • Header Neuzeit Grotesk ExtCond Black Two-tone — "SELF CARE" + "IN 3 SIZES"140pt
  • Sub Poppins Medium Frames the use case56pt
  • Descriptor (zone labels) Poppins Medium Short — "Heel", "Toe"50pt

  • Grid cells without measurements. Every cell needs its dimension — it's the purchase driver.
  • Arrows that point at nothing specific. Land each arrow on the exact body zone.
  • More than 4 grid cells. 2×2 max — beyond that, it clutters at mobile.
  • Lifestyle photo missing. The grid alone is functional. The lifestyle photo adds the warmth that converts.

Product Details

What's in the box. The dominant element is a giant quantity number — 10x, 6x, 3x. The customer leaves knowing exactly what they're getting.

  • Quantity number too small. Scale until it's visually equal to or larger than the product image.

  • Header longer than 4 words. "HAPPY HEALING" works at 212pt. "YOUR COMPLETE BLISTER KIT" wraps badly.

06
Brand signatures

The two-tone header rule

The single most recognizable element of every DFO carousel image. Mandatory at 140pt+ — no exceptions. The accent color always lands on the word carrying the emotional or informational weight.

140pt threshold
The rule. Any header at 140pt or larger MUST split across two colors. The first word(s) sit in near-black (#231f20) or deep navy (#1d396a) — the setup, the frame. The emphasis word(s) switch to the product line's accent color — the punchline, the claim. The break point is the natural language emphasis, never arbitrary. On dark backgrounds, the base word shifts to cream (#fef5e3) or white; the accent word may shift to a brighter variant for contrast.

Pulled from real headers across the 11 templates. Notice how the break always lands on the natural emphasis — not the middle of a phrase.

What this rule is NOT. It's not arbitrary — never split mid-word or mid-phrase awkwardly. It's not optional above 140pt — a single-color header at large size is not a DFO image. And it does not apply below 140pt — smaller text (subs, descriptors, labels) is always single-color.

9 path variants

3-Sparkle Cluster
2-color · per palette

Three 4-pointed stars in a constellation: large primary (lower center), medium secondary (upper left), small tertiary (upper right). Three sizes, two hues from the same accent family. Use Case A may use a 3-color variant.

Drop Shadow
Soft · 20–35% opacity

Soft shadow beneath product cutouts. Blur 20–40px, downward and slightly right, dark neutral (never colored). Mandatory in Feature Dominant when product tone matches the background.

Pop-Out Circle
Use Case B only

Circular crop of the product's most compelling detail — grip texture, gel material, weave. Thick accent-color outline, connecting arrow in the exact same hex. Must be revelatory — never a generic crop.

Used inUse Case B exclusively

Badges
4–6 max · white on color

Small rounded shapes — white line icon on a complementary (not identical) accent fill. Each badge = one benefit, with a Poppins SemiBold label beneath. Stack vertically, aligned to one edge.

07 · Anchor label
Tab Feature
Small · 1–3 words

Rounded rectangle with a pointed end — file-folder tab shape. Labels a variant, SKU, or attribute adjacent to its content without needing a callout line. Never larger than the text it labels.

Used inProduct Details primarily · Use Case B
08 · Authority
Signature Block
Handwritten · not calligraphy

Three parts: handwritten script signature ("— Dr. Frederick"), printed credentials in Poppins SemiBold ("Ben Frederick, MD"), and a portrait. Sharpie-feel, not designed. Business-letter spacing between lines.

Day-Count Badge

Compact pill — Poppins SemiBold white text on the product accent fill. Specific and honest: "1 Day", "6 Days", "4 Weeks". Overlays the corner of a before/after photo. Counts must reflect real typical outcomes.

Connecting Arrows
Two types · always colored

Type A: step navigation in How It Works — exact same hex as the header accent word. Type B: annotation arrows in Use Case A/B — colored, hand-indicated, never gray. Gray reads as inactive; arrows are directional signals.

The Judges

Three checks before save

Three named judges observe every L4 output before it saves to the carousel. Each judge has a defined scope. Fail any one → revise and regenerate.

✓

Truth

Before/after photos are real customer imagery — not retouched or theatrical. Day counts are accurate and reflect typical outcomes. Press logos are only used for actual earned coverage. Measurements are correct. Sales milestones are verified.

≡

Alignment

Two-tone header is present at correct size threshold (140pt+). Wave shape appears in the correct zone. Sparkle graphic is present and correctly sized. Product is visible in every image. No template mixing — each slot has one job.

★

Brand

Correct accent palette for the SKU family. DFO column is visually dominant in comparison charts. No fabricated claims. Media logos are not colorized to the product palette. Arrow color matches header accent exactly. Disclaimer present when required.

⇣

Tracing failures. If a judge rejects an output, the trace always leads back to a specific layer. Truth failures → trace to L0/L1 product facts. Alignment failures → trace to L2 template .md file. Brand failures → trace to L1 color-system.md or L2 template rules.
